# Getting Started

This guide walks you through your first session in SGM, from logging in to navigating the main features.

## Logging In

1. Navigate to your SGM instance URL
2. Enter your email address
3. Complete authentication (SSO or password, depending on your organization's setup)

<Info>
If you're using the demo environment, you can sign in with any email address. Demo mode uses synthetic data for evaluation.
</Info>

## Understanding the Dashboard

After logging in, you see the main dashboard with four operational mode cards:

| Mode | Color | Purpose |
|------|-------|---------|
| **Design** | Blue | Build policies and frameworks |
| **Operate** | Green | Daily operations and approvals |
| **Dispute** | Red/Orange | Case and exception handling |
| **Oversee** | Purple | Executive oversight and compliance |

Each card shows:
- Mode description
- Key metrics for that mode
- Quick access button

## Navigating Modes

### Switching Between Modes

1. Click any mode card on the dashboard to enter that mode
2. Use the mode selector in the navigation bar to switch modes
3. Click the dashboard icon to return to the main view

### Mode-Specific Navigation

Once inside a mode, you'll see:
- **Left sidebar**: Mode-specific navigation links
- **Main content area**: Current page content
- **Right panel**: Context-sensitive details (when applicable)

## The Dock

At the bottom of every page, you'll find the **dock** with quick-access features:

| Orb | Function |
|-----|----------|
| **Ask** | Query governance policies with AI |
| **Ops** | View operational alerts and patterns |
| **Pulse** | Get coaching recommendations |
| **Task** | Manage your work queue |
| **KB** | Search the Knowledge Base |

Click any orb to open its interface.

## Your First Tasks

### If You're a Policy Designer

<Steps>
  <Step title="Enter Design Mode">
    Click the Design mode card on the dashboard
  </Step>
  <Step title="Browse the Policy Library">
    Navigate to Policies to see all 17+ governance policies
  </Step>
  <Step title="Review a policy">
    Click any policy (try SCP-001 Clawback and Recovery) to see its full content
  </Step>
  <Step title="Check the Framework">
    Visit Governance Framework to see how policies map to governance pillars
  </Step>
</Steps>

### If You're in Operations

<Steps>
  <Step title="Enter Operate Mode">
    Click the Operate mode card on the dashboard
  </Step>
  <Step title="Check Approvals">
    Navigate to Approvals to see pending items requiring review
  </Step>
  <Step title="Review Documents">
    Visit Documents to see governance documents and their status
  </Step>
  <Step title="Check Notifications">
    Review any notifications for items needing your attention
  </Step>
</Steps>

### If You Handle Disputes

<Steps>
  <Step title="Enter Dispute Mode">
    Click the Dispute mode card on the dashboard
  </Step>
  <Step title="View Active Cases">
    Navigate to Cases to see the current case queue
  </Step>
  <Step title="Review a case">
    Click any case to see its details, timeline, and resolution options
  </Step>
  <Step title="Check SLA Status">
    Visit Cases > SLA to see time-sensitive cases
  </Step>
</Steps>

## Asking Questions with AI

SGM includes an AI assistant that understands your governance policies:

1. Click the **Ask** orb in the dock
2. Type your question in natural language, for example:
   - "What is the clawback policy for terminated reps?"
   - "How long do we have to process an exception request?"
   - "What approvals are needed for windfall deals?"
3. Review the response, which includes:
   - Direct answer to your question
   - Policy references and citations
   - Related policies or documents

## Searching Content

Use the global search (keyboard shortcut: `Cmd+K` or `Ctrl+K`) to find:
- Policies by name or content
- Documents by title or metadata
- Cases by ID or description
- Knowledge Base cards by topic
