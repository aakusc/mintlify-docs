# Policy Library

The Policy Library is the central repository for all governance policies in SGM. It contains 17 Sales Compensation Policies (SCPs) plus custom policies, covering all aspects of compensation governance.

## Accessing the Policy Library

**Path:** `/policies`

## Browsing Policies

### List View

The policy list shows:
- Policy code and name
- Category
- Status (Draft, Template, Approved)
- Word count
- Last updated date

### Filtering

Filter policies by:
- **Status**: Draft, Template, Approved
- **Category**: Any of the 9 categories
- **Search**: Policy name, code, or content

### Sorting

Sort by:
- Code (A-Z or Z-A)
- Name
- Category
- Last updated
- Word count

## Viewing a Policy

Click any policy to see its full content:

### Policy Header

- Policy code and name
- Status badge
- Category
- Word count
- Version number
- Last updated date

### Policy Content

Each policy includes:

| Section | Description |
|---------|-------------|
| **Purpose** | Why this policy exists |
| **Scope** | Who and what it applies to |
| **Policy Statement** | The core policy rules |
| **Procedures** | How to implement the policy |
| **Exceptions** | When exceptions apply |
| **Definitions** | Key terms explained |
| **Related Policies** | Links to related governance |

### Actions

From the policy detail view:
- **Download**: Export as Markdown
- **Copy Link**: Share the policy URL
- **View History**: See version changes
- **Edit** (if permitted): Modify the policy

## Policy Statuses

| Status | Meaning | Who Can Edit |
|--------|---------|--------------|
| **Draft** | Work in progress | Authors |
| **Template** | Standard template for reuse | Admins |
| **Approved** | Official, in effect | Requires workflow |

## Creating a Policy

<Note>
Policy creation requires ADMIN or SUPER_ADMIN role.
</Note>

<Steps>
  <Step title="Click Create Policy">
    Use the + button in the policy list toolbar
  </Step>
  <Step title="Select a template">
    Choose a base template or start blank
  </Step>
  <Step title="Fill in details">
    Add policy name, category, and content
  </Step>
  <Step title="Save as draft">
    Save to continue editing later
  </Step>
  <Step title="Submit for approval">
    When ready, submit to the approval workflow
  </Step>
</Steps>

## Policy Versioning

SGM tracks all policy changes:

- Each edit creates a new version
- Compare any two versions side-by-side
- Revert to previous versions if needed
- Audit trail shows who changed what and when

## Using Policies

### In Governance Analysis

When you run a [gap analysis](/user-guide/governance-analysis), results reference specific policies. Click any policy reference to see the full policy text.

### In Plan Templates

[Plan templates](/user-guide/plan-management) can incorporate policy references, ensuring plans comply with governance requirements.

### In Approvals

[Approval workflows](/user-guide/approvals) often reference policies to provide context for decisions.
